ratecurves.accumulate_index(arr, ind, op)#
Parameters:

  • arr (np.ndarray) – Array for the particular indexing.

  • ind (np.array, dtype=int or list) – List of integers for the indexing.

  • op (str) – Operation for the cumulative operation. Either “sum” or “prod”.

Returns:

Returns the same array but in the last dimension an indexing has occured and the missing positions has been “accumulated” in the next remaining position.

Return type:

np.ndarray

Examples

>>> arr = np.full((2, 8), 2)
>>> accumulate_index(arr, [0, 1, 2, 4, 7], "prod")
np.array([[2., 2., 2., 4., 8.],
          [2., 2., 2., 4., 8.]])
>>> arr = np.full((2, 8), 1)
>>> accumulate_index(arr, [0, 1, 2, 4, 7], "sum")
array([[1, 1, 2, 3],
       [1, 1, 2, 3]], dtype=int32)
ratecurves.number_to_array(*args, for_broadcasting=False)#

Convert the input arguments to NumPy arrays.

Parameters:

  • *args (scalar or array_like) – Input arguments to be converted to NumPy arrays.

  • for_broadcasting (bool, optional) – If True, reshape the arrays for broadcasting. Default is False.

Returns:

A list of NumPy arrays where each input argument is converted to an array. If an input argument is already a NumPy array, it is returned as is.

Return type:

list of ndarrays

Examples

>>> import numpy as np
>>> number_to_array(1.0, 2, np.array([3.0, 4.0]), for_broadcasting=True)
[array([[[1.]]]), array([[[2]]]), array([[[3.]],

       [[4.]]])]

Note

We need to add the BLANKLINE for format considerations, https://docs.python.org/3/library/doctest.html.
